About

Database

This database provides insider holdings and transactions through a simple and easy to access interface.

Coverage:

  • More than 15,000 issuers, and 200,000 insiders
  • More than 12 years of history

Data is sourced from SEC form 3, 4 & 5 filings.

See the examples and get started now.

Publisher

Sharadar is an independent research and analytics firm founded in 2013. Sharadar specializes in extraction, standardization and organization of financial data from company filings. We combine people, software and rigorous review processes to generate accurate, professional grade data for professional investors and analysts.

Getting Started

Getting Started

We'll demonstrate how to access the database, using filters to specify which data to retrieve. We'll start off with a compund example that uses multiple filters, then a series of simple examples that you can combine to your requirements in order to produce more complex queries.

1. Compound Example

The following request will retrieve the non-derivative holdings and transactions of Tim Cook in AAPL and NKE for the filing dates between 2014-12-31 and 2016-12-31 (inclusive) where the transaction value is greater than $100,000:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?api_key=4REPYvYN6PkUD57Cgy31&securityadcode=N,NA,ND&ownername=COOK TIMOTHY D&ticker=AAPL,NKE&filingdate.gte=2014-12-31&filingdate.lte=2016-12-31&transactionvalue.gte=100000

8. Operators

Data can retrieved based on greater than (gt), less than (lt), greater than or equal to (gte) or less than or equal to (lte) operators.

For example, the following request retrieves all specified ticker data where the filing date is greater than or equal to 2014-12-31:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?api_key=4REPYvYN6PkUD57Cgy31&ticker=AAPL&filingdate.gte=2014-12-31

The operators can be applied to the filter columns: ticker, filingdate, ownername, transactionvalue, and securityadcode.

9. Transactions above or below a specified threshold

Above a $100m threshold:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?api_key=4REPYvYN6PkUD57Cgy31&transactionvalue.gte=100000000

Below a -$100m threshold:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?api_key=4REPYvYN6PkUD57Cgy31&transactionvalue.lte=-100000000

10. Indicators (and Ticker)

You can limit the indicators (columns) retrieved. The following example will return the names of TSLA insiders, the dates of their SEC insider filing, their officer title (if applicable), and whether or not they are an officer, and/or a director:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?api_key=4REPYvYN6PkUD57Cgy31&ticker=TSLA&qopts.columns=ownername,filingdate,officertitle,isofficer,isdirector

11. Most Recent N Observations

Data is stored sorted in descending order by date of filing with the SEC. Therefore to retrieve the most recent N items filed with the sec you can utilize the qopts.per_page parameter. In this example we will retrieve the most recent 100 items:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?api_key=4REPYvYN6PkUD57Cgy31&qopts.per_page=100

If unspecified the default number of rows per call is 10,000.

Batch Download

Subscribers can download the entire database at anytime. First, access the following url:

https://data.nasdaq.com/api/v3/datatables/SHARADAR/SF2.csv?qopts.export=true&api_key=4REPYvYN6PkUD57Cgy31

This will return csv text that includes a file status code. If the status code is "creating", "generating" or "regenerating" then wait a minute or two and try again. When the status code is "fresh" then use the url that is also returned in the csv text in order to retrieve the bulk download.

In addition you can add other arguments to filter the data that is retrieved through this batch process, as described in the docs.

Notes and Issues

  • data is sourced from SEC form 3, 4 and 5.

  • The SharesOwnedBeforeTransaction and SharesOwnedFollowingTransaction are as reported in the underlying SEC filings. There is some complexity to them which it is necessary to bear in mind. At a minimum these fields represent separate sub-totals for each of derivative and non-derivative holdings, identifiable through the SecurityADCode field. Some filers segement this further to represent subtotals for DirectOrIndirect holdings and / or SecurityTitle.

  • data is currently not adjusted for stock splits.

  • where a filing has been subsequently restated the FormType field of the restated filing will be prepended with "RESTATED".
