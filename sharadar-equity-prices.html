        About

        Database

        Updated daily, this database provides End-Of-Day (EOD) company prices, corporate actions, and reference data including:

        • Open, high, low, close, and volume (OHLCV) data
        • Unadjusted prices, prices adjusted for stock splits, and prices adjusted for both stock splits and stock dividends
        • Detailed corporate actions information such as splits, dividends, spinoffs, acquisitions, delist reasons, and ticker changes
        • Reference information covering a variety of classifications and security identifiers

        Coverage represents a superset of the common stock companies covered in Sharadar's Fundamental dataset, SF1. In addition, alternative classes of stock are included as well as prefered stock and warrants.

        More than 21,000 active and delisted tickers are currently covered, with history to the year 1998. Coverage is continuously expanding.

        Favourable pricing is provided in our bundle offerings here and here.

        Data is updated at 5:00pm and 11:00pm ET every business day.

        Publisher

        Sharadar is an independent research and analytics firm founded in 2013. Sharadar specializes in extraction, standardization and organization of financial data from company filings. We combine people, software and process to generate accurate, professional grade data for professional investors and analysts.

        Getting Started

        Getting Started

        Let's get started with a few basic examples of how to access the data using the various tools that are available to you: the Web API, Python, R, and Excel. These simple examples can be combined to produce more complex queries.

        There are 5 tables associated with this database. You can see these tables visually presented here. The primary table is SHARADAR/SEP. The other tables are SHARADAR/ACTIONS, SHARADAR/TICKERS, SHARADAR/METRICS and SHARADAR/INDICATORS.

        In each table data are presented in tabular format with indicators represented in columns.

        When you query data from a particular table you can specify "filters" which limit the data that is retrieved. If you do not specify a filter the entire table, all rows and columns, will be returned but limited to 10,000 rows.

        There are 3 filter columns in the primary SHARADAR/SEP table that you can use to specify which data to retrieve, these are: ticker, date and lastupdated.

        When you integrate this API you should not assume a particular column order as additional indicators will be added in the future.

        Data is not sorted by default and you should sort the data upon retrieval.

        In these examples we will assume that you have the associated package installed where necessary. In the case of the Web API, no Nasdaq Data Link package needs to be installed. You can find installation instructions and generic documentation for each of the tools here. The data retrieval methods described here are referred to under the "tables" section of Nasdaq Data Link's generic documentation, rather than the "time-series" section.

        1. Authentication

        All requests must be authenticated with your API key which can be found in account settings, here.

        Web API: include your API key in the URL as shown in the examples to follow

        Python: See our python documentation for configuring your API Key.

        R: Quandl.api_key<"YOURAPIKEY">

        Excel: include your API key in the Quandl tab on Excel's ribbon

        2. Ticker

        To retrieve all data for a specified ticker:

        Web API: https://data.nasdaq.com/api/v3/datatables/SHARADAR/SEP.csv?ticker=AAPL&api_key=4REPYvYN6PkUD57Cgy31

        Python: nasdaqdatalink.get_table('SHARADAR/SEP', ticker='AAPL')

        R: Quandl.datatable('SHARADAR/SEP', ticker='AAPL')

        Excel: QTABLE("SHARADAR/SEP", , "ticker", "AAPL")

        3. Multiple Tickers

        To retrieve data for multiple specified tickers:

        Web API: https://data.nasdaq.com/api/v3/datatables/SHARADAR/SEP.csv?ticker=AAPL,TSLA&api_key=4REPYvYN6PkUD57Cgy31

        Python: nasdaqdatalink.get_table('SHARADAR/SEP', ticker=['AAPL','TSLA'])

        R: Quandl.datatable('SHARADAR/SEP', ticker=c('AAPL', 'TSLA'))

        Excel: QTABLE("SHARADAR/SEP", , "ticker", {"AAPL","TSLA"})

        5. Date Range

        To retrieve data for a specified data range:

        Web API: https://data.nasdaq.com/api/v3/datatables/SHARADAR/SEP.csv?date.gte=2017-01-01&date.lte=2017-10-30&ticker=AAPL&api_key=4REPYvYN6PkUD57Cgy31

        Python: nasdaqdatalink.get_table('SHARADAR/SEP', date={'gte':'2017-01-01', 'lte':'2017-10-30'}, ticker='AAPL')

        6. Indicators (Columns)

        To retrieve only specific indicators (columns):

        Web API: https://data.nasdaq.com/api/v3/datatables/SHARADAR/SEP.csv?qopts.columns=ticker,date,open,high,low,close,volume,closeadj&api_key=4REPYvYN6PkUD57Cgy31

        Python: nasdaqdatalink.get_table('SHARADAR/SEP', qopts={"columns":['ticker' ,'date','open','high','low','close','volume','closeadj']}, ticker='AAPL')

        Excel: QTABLE("SHARADAR/SEP", {"ticker","date","open","low","close","volume","closeadj"}, "ticker", "AAPL")

        FAQs

        Frequently Asked Questions

        What corporate adjustments are applied?

        The following corporate adjustments are applied to the fields listed below:

        Open, High, Low, Close and Volume

        These fields are adjusted for stock splits and stock dividends. They are not adjusted for cash dividends or spinoffs.

        CloseAdj

        This field is adjusted for stock splits, stock dividends, cash dividends and spinoffs.

        CloseUnadj

        This field is unadjusted and does not apply corporate adjustments for stock splits, stock dividends, cash dividends or spinoffs..

        For adjusted price fields that are not included in this product (e.g., OpenAdj), they can be easily imputed by calculating adjusted/unadjusted fields.

        For example, OpenAdj can be calculated as following: OpenAdj = Open * CloseAdj / Close.

        For further detail of corporate actions such as splits, spinoffs, and dividends, please see the Corporate Actions table SHARADAR/ACTIONS.
